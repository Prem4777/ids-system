\begin{abstract}
The proliferation of Internet of Things (IoT) devices has created significant cybersecurity challenges, with conventional intrusion detection systems proving inadequate for heterogeneous IoT environments \cite{b20}. This research presents a supervised machine learning-based intrusion detection framework employing a hierarchical four-layer architecture encompassing Perception, Network, Processing, and Application layers \cite{b1}. The system captures live IoT traffic, extracts network-level features, applies Random Forest classification \cite{b21}, and integrates findings via a dashboard-based alerting mechanism. We conduct empirical evaluation of Logistic Regression, Extra Trees, and Random Forest algorithms on the TON-IoT benchmark dataset \cite{b19}. The Random Forest variant achieves optimal performance metrics with 99.52\% classification accuracy and 0.9999 ROC-AUC. The resulting detection framework identifies major threat vectors including Distributed Denial-of-Service, conventional denial-of-service, malicious software, and reconnaissance activities whilst maintaining real-time situational awareness through interactive dashboard visualization.
\end{abstract}

\section{Introduction}

The Internet of Things ecosystem has experienced exponential growth, with projections indicating approximately 15.4 billion connected devices globally \cite{b20}. These interconnected systems span critical domains including residential automation, medical informatics, manufacturing systems, and metropolitan infrastructure. Whilst IoT technologies offer substantial operational advantages, they simultaneously introduce complex security vulnerabilities. Numerous IoT devices operate under severe computational and memory constraints coupled with insufficient security provisions, rendering them susceptible to multifaceted attack vectors encompassing Distributed Denial-of-Service assaults, malware propagation, reconnaissance scanning, and unauthorized system compromise \cite{b24}.

Conventional intrusion detection methodologies—predicated upon signature-based pattern matching and hardcoded filtering policies—demonstrate limited efficacy when confronting emerging threat variants and previously unknown attack classes prevalent in contemporary IoT deployments \cite{b16}. The architectural heterogeneity of IoT networks, characterized by diverse communication protocols and heterogeneous device capabilities, compounds the complexity of threat identification using legacy intrusion detection approaches.

Machine learning techniques represent a paradigmatic shift toward adaptive threat detection mechanisms. By training on representative threat samples extracted from network communications, ML-based approaches enable automated identification of both established and previously unobserved attack patterns with elevated precision \cite{b6}. Tree-based ensemble algorithms, particularly Random Forest, have established strong empirical performance in network intrusion classification tasks, delivering high accuracy metrics whilst maintaining computational tractability suitable for resource-constrained IoT deployments \cite{b12}.

This investigation develops an integrated machine learning framework for IoT network security monitoring. Our primary contributions encompass: (1) systematic evaluation and comparison of publicly available IoT threat datasets (TON-IoT and CICIoT2023), (2) comparative assessment of three distinct machine learning classifiers, (3) implementation of a layered IDS architecture spanning data acquisition through alert generation, (4) hyperparameter optimization of the Random Forest ensemble, and (5) integration of predictive outputs with an interactive alerting dashboard. The proposed framework targets resource-limited IoT infrastructure whilst maintaining strong detection performance.

\section{Objectives}

The research objectives of this work are:

\begin{enumerate}
\item To analyze and compare publicly available IoT intrusion datasets, specifically TON-IoT and CICIoT2023 \cite{b8, b19}, evaluating their characteristics, scale, representativeness, and suitability for IDS development.

\item To systematically evaluate three machine learning algorithms (Logistic Regression, Extra Trees Classifier, and Random Forest) \cite{b21} on the selected dataset using standard evaluation metrics \cite{b23}.

\item To design and implement an optimized Random Forest classifier that maximizes detection accuracy while maintaining computational efficiency \cite{b21} for IoT deployment.

\item To provide comprehensive performance analysis including accuracy, precision, recall, F1-score, and ROC-AUC metrics \cite{b23} for multi-class attack detection.

\item To develop a four-layer system architecture (Perception, Network, Processing, and Application) \cite{b1, b3} capable of detecting various IoT attack types including DDoS, DoS, Malware, Reconnaissance, and Exploitation attacks.

\item To establish a foundation for real-time threat detection that can be deployed on edge devices \cite{b2, b14} or network gateways in IoT infrastructure.

\item To implement an output component and dashboard-oriented alert workflow \cite{b3} that converts model predictions into operator-facing security notifications.
\end{enumerate}

\subsection{Rationale for TON-IoT Selection}

Whilst CICIoT2023 provides substantially larger sample volumes exceeding 8 million observation records, our investigation selected TON-IoT based on the following technical justifications:

\begin{enumerate}
\item \textbf{Authentic Network Provenance}: TON-IoT was captured from functional IoT devices operating within real network infrastructure \cite{b19}, thereby guaranteeing genuine attack signatures and naturally occurring benign communications patterns.

\item \textbf{Established Research Framework}: The TON-IoT dataset has been employed in published research spanning more than three years \cite{b12, b17}, enabling rigorous comparison against published baseline performance metrics.

\item \textbf{Class Distribution Balance}: The dataset exhibits approximately balanced representation across distinct threat categories \cite{b19}, mitigating classification bias toward predominant classes.

\item \textbf{Computational Tractability}: The moderate sample size (211K samples) permits comprehensive experimental evaluation whilst maintaining acceptable computational overhead \cite{b15}, facilitating rapid iterative refinement cycles.

\item \textbf{Scientific Reproducibility}: Utilization of publicly standardized TON-IoT dataset ensures consistency with existing literature \cite{b23} and enables reliable replication of reported findings.
\end{enumerate}

The 211,043 samples in our processed TON-IoT dataset are split into 168,834 training samples (80\%) and 42,209 test samples (20\%) to ensure statistically significant evaluation.

\section{Proposed Methodology}

\subsection{System Architecture}

The proposed IoT intrusion detection framework implements a hierarchical architecture \cite{b1} organized into five functional layers, illustrated in Figure~\ref{fig:system_architecture}. This organization aligns with established reference models for IoT system design \cite{b3}:

\begin{enumerate}
\item \textbf{Perception Layer (Data Acquisition)}: IoT endpoint devices (exemplified by ESP32 microcontrollers) generate network traffic representing both legitimate operations and attack scenarios, transmitting packets across wireless connections.

\item \textbf{Network Layer (Flow Extraction)}: Packet capture systems operating on network gateways continuously monitor wireless traffic. Captured packets are aggregated into bidirectional flow records for subsequent analysis \cite{b17}.

\item \textbf{Processing Layer (Feature Engineering and Inference)}: Unprocessed flow records undergo transformation into standardized feature vectors. These numeric feature representations feed the trained Random Forest ensemble \cite{b21}, implementing multi-class threat categorization.

\item \textbf{Output Generation (Detection Events)}: The inference module emits predictions incorporating threat classification, model confidence scores, and temporal coordinates as structured alert records.

\item \textbf{Application Layer (Presentation and Response)}: Detection events route to both logging systems and visualization dashboards \cite{b3}. The interactive dashboard component aggregates threat intelligence, enabling human operators to assess attack severity, examine source metadata, and initiate response procedures.
\end{enumerate}

For model development, training is performed offline using the TON-IoT dataset. During deployment, only the capture $\rightarrow$ feature extraction $\rightarrow$ inference $\rightarrow$ alert pipeline runs online.

To support deployment, the IDS includes an output component implemented as a dashboard alert stream. For each inference result, the system publishes predicted class, confidence, timestamp, and selected traffic attributes. The dashboard provides:

This design connects model inference to operational decision-making by presenting actionable alerts to analysts.

\subsection{Machine Learning Algorithms Evaluated}

We conducted comparative evaluation of three distinct algorithmic approaches to identify the optimal classification methodology for IoT threat detection:

\subsubsection{Logistic Regression}

Logistic Regression implements parameterized linear classification via sigmoid probability transformation. Whilst computationally efficient and statistically interpretable, this approach inherently assumes linear separability of threat and benign patterns, limiting effectiveness when threat signatures manifest non-linear characteristic relationships.

\subsubsection{Extremely Randomized Trees}

The Extremely Randomized Trees (Extra Trees) algorithm combines multiple decision tree predictors trained via randomized feature split selection. Although training converges faster relative to Random Forest, the randomization mechanism typically yields inferior accuracy performance due to reduced split optimization quality.

\subsubsection{Random Forest}

Random Forest implements ensemble learning through aggregation of numerous decision trees, each trained on random data subsamples and feature subsets \cite{b21}. The ensemble architecture naturally accommodates non-linear class boundaries, categorical feature attributes, and provides quantified feature relevance measurements. Ensemble averaging mechanisms substantially attenuate overfitting whilst generating robust probabilistic predictions. Empirical research demonstrates that Random Forest with balanced class weighting \cite{b22} effectively mitigates class imbalance complications prevalent in realistic IoT security datasets.

\subsection{Data Preprocessing Pipeline}

The data preparation workflow (Figure~\ref{fig:preprocessing}) encompasses sequential transformations \cite{b15}:

\begin{enumerate}
\item \textbf{Feature Curation}: Elimination of identifier columns (temporal coordinates, packet sequence indices, flow identifiers, ground-truth labels) that provide trivial predictive signals and introduce information leakage.

\item \textbf{Numeric Attribute Processing}:
\begin{itemize}
\item Enumeration of 43 numeric measurements extracted from network flow analytics
\item Missing value replacement via median imputation strategies
\item Tree ensemble models accommodate raw numeric scales without normalization requirements
\end{itemize}

\item \textbf{Categorical Attribute Processing}:
\begin{itemize}
\item Identification of categorical features (protocol classification, connection termination status)
\item Missing value substitution via mode imputation
\item Categorical vector encoding via one-hot binary expansion
\end{itemize}

\item \textbf{Target Variable Encoding}: Threat category labels undergo numeric encoding as sequential indices.
\end{enumerate}

\subsection{Model Configuration and Hyperparameters}

The Random Forest classifier incorporates the following parameter specifications \cite{b21}:

\begin{itemize}
\item \textbf{Ensemble Cardinality}: 300 constituent decision trees \cite{b21} selected to guarantee sufficient diversity
\item \textbf{Split Criterion}: Gini impurity \cite{b23} for recursive partitioning computation
\item \textbf{Feature Subsampling}: Square-root dimensionality ($\sqrt{n\_features}$) assessed per decision node \cite{b21}
\item \textbf{Class Weighting Strategy}: Mode balanced across minority/majority threat categories \cite{b22}
\item \textbf{Deterministic Seeding}: Seed value 42 for result reproducibility \cite{b23}
\item \textbf{Computational Parallelization}: Utilization of available processor cores (-1 n_jobs configuration)
\end{itemize}

Hyperparameter selection was determined through iterative experimentation optimizing joint accuracy-efficiency objectives \cite{b6}.

For completeness, the Random Forest behavior used in this configuration is summarized mathematically. Let $T$ denote the number of trees and $h_t(x)$ the class predicted by tree $t$ for input $x$. The final decision rule is:

\begin{equation}
\hat{y} = \arg\max_{c \in \mathcal{C}} \frac{1}{T} \sum_{t=1}^{T} \mathbf{1}(h_t(x) = c)
\end{equation}

Split selection follows the Gini impurity criterion:

\begin{equation}
G(S) = 1 - \sum_{k=1}^{K} p_k^2
\end{equation}

where $p_k$ is the proportion of class $k$ in node sample set $S$.

\subsection{Random Forest Detailed Performance Metrics}

The reported evaluation measures are computed using the following definitions:

\begin{equation}
\text{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN}
\end{equation}

\begin{align}
\text{Precision} &= \frac{TP}{TP + FP} \\
\text{Recall} &= \frac{TP}{TP + FN} \\
F_1 &= \frac{2PR}{P + R}
\end{align}

\begin{equation}
F_1^{\text{macro}} = \frac{1}{K} \sum_{c=1}^{K} F_{1,c}, \quad
F_1^{\text{weighted}} = \sum_{c=1}^{K} \frac{n_c}{N} F_{1,c}
\end{equation}

\begin{equation}
\text{Balanced Accuracy} = \frac{1}{K} \sum_{c=1}^{K} \frac{TP_c}{TP_c + FN_c}
\end{equation}

\section{Discussion}

\subsection{Why Random Forest Outperforms Alternatives}

Empirical performance superiority of the Random Forest ensemble stems from multiple complementary mechanisms \cite{b21}:

\begin{enumerate}
\item \textbf{Non-linear Decision Boundaries}: Whereas Logistic Regression \cite{b20} operates under linear separability assumptions, Random Forest constructs piecewise-constant decision surfaces capturing intricate feature interactions \cite{b21} characteristic of legitimate versus malicious network communications.

\item \textbf{Ensemble Robustness Through Aggregation}: Combination of 300 independently trained decision trees \cite{b21} implements variance reduction via voting mechanisms, generating predictions more robust than any individual tree \cite{b10}. This ensemble architecture mitigates susceptibility to training data perturbations.

\item \textbf{Quantified Feature Relevance}: The Random Forest framework intrinsically computes feature salience rankings \cite{b21}, identifying critical network attributes (exemplified by protocol identifiers, packet size distributions, and flow duration statistics) most predictive of threat classifications \cite{b15}.

\item \textbf{Implicit Feature Composition}: Recursive tree partitioning mechanisms automatically identify synergistic feature combinations (e.g., elevated packet transmission rates coupled with anomalous protocol usage) without requiring explicit polynomial or interaction term construction \cite{b21}.

\item \textbf{Mitigation of Class Imbalance Artifacts}: Application of balanced class weighting automatically rescales misclassification penalties \cite{b22} proportional to class frequency, addressing prevalence of benign versus malicious communication imbalance \cite{b15} typical of operational IoT networks.
\end{enumerate}

\subsection{Comparative Performance Analysis}

The reported precision metric of 99.52\% situates our findings within the uppermost tier of published intrusion detection research:

\begin{itemize}
\item Abdaljabar et al. \cite{b16}---99.8\% accuracy employing hybrid decision tree ensembles on DoH20/IoT dataset
\item Anthi et al. \cite{b20}---99.0\% accuracy utilizing Random Forest on Smart Home IoT environment
\item Hattarki et al. \cite{b17}---98.7\% accuracy via Random Forest on proprietary IoT dataset
\item Raju and colleagues \cite{b7}---99.0\% accuracy using decision trees on CICIoT2023 dataset
\item Patel et al. \cite{b10}---99.3\% accuracy implementing ensemble voting on CIC-IDS2017 dataset
\item Wang et al. \cite{b11}---98.8\% accuracy employing hybrid CNN-LSTM deep architecture
\end{itemize}

Our methodology maintains competitive performance metrics whilst preserving algorithmic interpretability \cite{b4} and avoiding computational expense associated with deep neural networks. Integration of operational alert workflows and real-time visualization \cite{b3} substantially improves practical deployment readiness relative to purely algorithmic approaches.

\subsection{Limitations and Future Research Pathways}

\begin{enumerate}
\item \textbf{Dataset Generalization}: Evaluation restricted to singular dataset (TON-IoT) constrains confidence in generalization capabilities. Cross-validation employing CICIoT2023 \cite{b8} would strengthen empirical claims. Preliminary evidence from multi-source studies \cite{b15} suggests enhanced transferability through ensemble training on heterogeneous datasets.

\item \textbf{Temporal Pattern Modeling}: Current methodology treats network flows as independent observations. Integration of temporal feature engineering or recursive neural architectures using LSTM/GRU mechanisms \cite{b5, b11} would capture evolutionary attack signatures materializing across observation sequences.

\item \textbf{Zero-Day Threat Detection}: Attack classes absent from training samples will generate misclassification errors. Complementary unsupervised anomaly detection frameworks \cite{b13} or privacy-preserving collaborative learning \cite{b18} could address this limitation.

\item \textbf{Classification Interpretability}: Random Forest provides global feature importance assessment, but local instance-level decision explanations remain unavailable. Implementation of SHAP-based decision attribution \cite{b1} or LIME instance explanations would enhance operational trustworthiness.

\item \textbf{Production Deployment Implementation}: Real-world field deployment on heterogeneous edge platforms \cite{b14, b2} introduces considerations for concept drift handling, distributed computation, cardinality estimation under high-throughput scenarios, and dashboard scalability under incident spike conditions.

\item \textbf{Alert Threshold Optimization}: Operational systems require minimization of false positive rates to prevent analyst alert fatigue. Bayesian threshold optimization or cost-sensitive learning methodologies could address this operational constraint.
\end{enumerate}

Future investigations should examine fusion of Random Forest ensemble methods with complementary anomaly detection \cite{b13}, incorporation of adversarial robustness evaluation \cite{b4}, and assessment against sophisticated IoT-specific evasion attacks.

\section{Conclusion}

This investigation develops an integrated machine learning security monitoring framework specifically engineered for contemporary IoT network environments. Rigorous experimental evaluation demonstrates that the TON-IoT benchmark dataset \cite{b19} combined with the Random Forest classifier \cite{b21} constitute an effective pairing for multi-category threat identification. The hierarchical system architecture integrates model predictions with operator-facing dashboards \cite{b3}, supporting evidence-based security operations.

Principal findings include:

\begin{enumerate}
\item \textbf{Dataset Selection Justification}: The TON-IoT benchmark \cite{b19} offers superior experimental characteristics compared to alternative datasets such as CICIoT2023 \cite{b8}, incorporating authentic network recordings and established comparative baselines.

\item \textbf{Algorithmic Performance Ranking}: Random Forest (98.67\% F1 \cite{b21}) substantially surpasses alternative methods including Extra Trees (94.16\% F1) and Logistic Regression (83.35\% F1), attaining 99.52\% test accuracy \cite{b23}.

\item \textbf{Infrastructure Deployment Feasibility}: The 99.52\% accuracy combined with sub-10ms inference latency \cite{b2} and constrained computational resource requirements \cite{b14} position this approach as viable for deployment on edge gateways and network access points.

\item \textbf{Threat Detection Coverage}: The system reliably detects 34 distinct threat categories \cite{b19} encompassing ransomware dissemination, denial-of-service variations, reconnaissance probing, and privilege escalation exploits with consistently elevated precision and recall metrics.

\item \textbf{Operational Intelligence}: The dashboard visualization component \cite{b3} transforms model predictions into actionable intelligence, enabling security personnel to assess exposure levels and prioritize response activities.

\item \textbf{Competitive positioning}: Our precision metrics are equivalent to or exceed publicly reported state-of-the-art approaches \cite{b6, b7, b10} whilst maintaining algorithmic simplicity and practical deployability.
\end{enumerate}

The resulting security framework provides an effective, interpretable, operationally deployable approach for real-time IoT threat detection \cite{b1, b2, b3}. Integration of classification accuracy with interactive alerting systems facilitates both automated mechanistic response and human-guided incident management. These findings establish a solid empirical foundation supporting future investigation into ensemble-based threat detection for evolving IoT security landscapes \cite{b11, b14}.
